# 🧭 Unknown Compass (언노운 나침반)

AI에게 일을 시키기 전에, **내가 모르는 것부터 찾는** 프롬프트 기반 자가 진단 도구.

**🌐 라이브 배포: <https://unknown-compass.vercel.app>** — 설치·가입 없이 브라우저에서 바로.
API 키가 없어도 [`#demo`](https://unknown-compass.vercel.app/#demo)로 예시 진단을, [`#log`](https://unknown-compass.vercel.app/#log)로 기록 추이를 즉시 볼 수 있다.

## 왜 만들었나

Anthropic Claude Code 팀의 Thariq가 쓴 *A Field Guide to Fable: Finding Your Unknowns*
([원문 글 (X)](https://x.com/trq212/status/2073100352921215386))의 핵심 주장:

> 이제 결과물의 품질은 모델의 성능이 아니라, **내 언노운(모르는 것)을 밝혀내는 능력**에 발목 잡힌다.

이 관찰은 우리 공모전 제안서의 대응 방안(AI 리터러시)과 정확히 맞닿아 있다.
AI 리터러시의 실체는 도구 사용법 교육이 아니라 **자기 작업에 대한 메타인지** —
내가 뭘 알고 뭘 모르는지 아는 능력이다. 이 도구는 그 능력을 즉시 연습할 수 있게
만든 프로토타입이다.

## 4가지 언노운 (원문 글에서 추출)

| | 지식: 안다 | 지식: 모른다 |
|---|---|---|
| **인식함** | **KK** 아는 것을 안다 — 프롬프트에 적어 전달하는 것 | **KU** 모른다는 걸 안다 — 답은 없지만 정해야 함은 인식 |
| **인식 못함** | **UK** 안다는 걸 모른다 — 당연해서 안 적지만 결과를 보면 "이게 아닌데" | **UU** 모른다는 것도 모른다 — 기준 자체를 모르는, 가장 위험한 영역 |

이 4분면 구도 자체는 [Rumsfeld Matrix](https://en.wikipedia.org/wiki/There_are_unknown_unknowns)로 알려진 프레임워크다 —
도널드 럼즈펠드 미 국방장관의 2002년 2월 12일 국방부 브리핑("known knowns / known unknowns / unknown unknowns")으로
대중화됐고, 개념적 기원은 심리학의 Johari Window(Joseph Luft & Harrington Ingham, 1955)까지 거슬러 올라간다.
"unknown knowns(UK)" 사분면은 원 발언에는 없던 것으로, 이후 매트릭스가 2×2로 정식화되며 채워진 영역이다.

심리학적 근거: Kruger, J., & Dunning, D. (1999). Unskilled and unaware of it.
*Journal of Personality and Social Psychology*, 77(6), 1121–1134.
<https://doi.org/10.1037/0022-3514.77.6.1121> — "모른다는 것을 모르는" 상태가
실제 인지 편향으로 측정된 연구로, 이 도구가 UK·UU 사각지대를 경보하는 이유다.

## 사용법

<https://unknown-compass.vercel.app> 으로 접속하거나 `index.html`을 브라우저로 직접 연다. `#demo` 해시로 열면 예시가 즉시 렌더링된다.

**기본 (원클릭 진단, OpenRouter BYOK):**
1. 우측 상단 ⚙ 설정에 OpenRouter API 키 입력 (브라우저 localStorage에만 저장, 저장소·서버 전송 없음)
2. 하단 입력 바에 평소 AI에 시키듯 작업을 적고 **Enter(전송)** — 내부에서 진단 프롬프트 생성 → 모델 호출 → JSON 파싱이 자동 진행되고, 화면 중앙 **4분면 나침반이 갱신**된다
3. 대화는 누적되지 않는다 — 입력은 전송 후에도 남아 있어, 고쳐서 다시 보내면 **재진단**되는 단발 진단 도구다
4. 기본 모델은 GLM 계열 저비용 모델 — 진단용 필터 LLM이라 고성능 thinking 모델이 필요 없다.
   설정을 열면 OpenRouter 공개 모델 목록 전체를 드롭다운으로 불러와 선택할 수 있다 (실패 시 기본 3종 폴백)

**수동 모드 (API 키 불필요, 폴백):**
하단 "API 키 없이 쓰기"를 펼치면 기존 플로우 그대로 — 진단 프롬프트 복사 → 아무 AI에나 붙여넣기 → JSON 응답을 붙여넣으면 나침반 갱신. 키가 없거나 호출이 실패하면 도구가 자동으로 이 모드를 안내한다.

각 항목에는 원문의 8가지 대응 패턴(블라인드스팟 패스 / 브레인스토밍·프로토타입 /
AI 역질문 / 레퍼런스 / 구현 계획서 / 구현 노트 / 피치 문서 / 퀴즈) 중 권장 패턴이 붙는다.
(원문의 "인터뷰" 패턴은 AI가 나에게 되묻게 하는 것이라는 뜻이 드러나도록 "AI 역질문"으로 표기.)
요약부는 인식 사각지대(UK+UU) 비율을 계산해 "지금 시켜도 되는지"를 경보한다.

## 진단 기록 (내 언노운 추이)

나침반 아래 **"📋 내 진단 기록"** 패널을 펼치면, 진단할 때마다 그 결과가
이 브라우저에만 자동 누적된다(최근 200건, `localStorage`, **서버 전송 없음**).

- **타임라인**으로 최신순 표시 — 회차별 사분면 분포 막대와 인식 사각지대(UK+UU) 비율,
  그리고 **직전 진단 대비 증감(▼/▲ %p)**을 함께 보여준다. 같은 작업을 고쳐 다시
  진단할수록 사각지대가 어떻게 줄어드는지가 한눈에 드러난다.
- **JSON / CSV 내보내기**로 기록 전체를 파일로 저장할 수 있다(CSV는 Excel 한글 대응).
- **전체 삭제**는 확인 배너("정말 삭제하시겠습니까?")를 거쳐 이 브라우저의 기록을 지운다.
- 키가 없어도 **[▶ 데모 로그 채우기]**(또는 `#log` 해시)로 예시 추이를 즉시 확인·조작할 수 있다.

이 기록은 대응 방안(AI 리터러시)의 실증부다 — **AI를 쓰는 행위 자체가 곧 자기 메타인지
훈련의 기록**이 되며, "사용 패턴이 리터러시의 증거"라는 주장을 도구가 직접 뒷받침한다.

## 설계 원칙

- **프롬프트 자체가 도구**: 진단 로직 전체가 사람이 읽을 수 있는 프롬프트로 노출된다
  (수동 모드에서 그대로 복사 가능) — 도구를 쓰는 것만으로 방법론이 학습되는 구조
  (AI 리터러시 도구가 블랙박스면 자기모순이므로). OpenRouter 연동은 같은 프롬프트를
  대신 호출해주는 편의 계층일 뿐이다.
- **단일 정적 HTML**: 의존성 0, 어디서나 동작, GitHub Pages 배포 가능
  (OpenRouter는 브라우저에서 직접 호출 — 서버 불필요). API 키는 localStorage에만 저장.
- **접근성**: 사분면 색상 팔레트는 라이트/다크 두 모드 각각 색각 이상(CVD) 분리도·명도 대비를
  검증했고, 색상만으로 의미를 전달하지 않는다(모든 사분면에 텍스트 라벨).
  디자인은 브루탈리스트 포스터 톤 — 기본은 시스템 설정을 따르고, 헤더의 테마 버튼으로
  자동/라이트/다크를 전환할 수 있다(선택은 localStorage에 저장).
  다크 모드는 라이트를 단순 반전하지 않고 Material 다크 테마 원칙을 따른다:
  순검정 대신 어두운 웜 그레이 배경, 고도는 더 밝은 서피스로 표현,
  순백 대신 오프화이트 텍스트, 액센트·상태색은 밝고 탈채도된 톤.
