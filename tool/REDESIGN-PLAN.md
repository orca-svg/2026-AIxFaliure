# 언노운 나침반 UI 개편 기획안

> 준엽의 개편 기획을 2026-07-10 팀 회의에서 공유·합의한 문서. 구현 전 기획 — 화요일(7/14) 취합 회의 전 크로스 체크용.

## 1. 배경·목적

언노운 나침반(`tool/index.html`)은 제안서 **3번(대응 방안)** 의 실증 프로토타입이다:
AI 리터러시의 실체는 교육과정이 아니라 **사용 패턴 속에서 강제되는 메타인지** — 내가
뭘 알고 모르는지(4분면)를 스스로 진단하는 능력이라는 주장을, 말이 아니라 동작하는
도구로 보여준다.

개편 방향(채팅형 단일 플로우 + 실시간 나침반 + 모델 연동)은 준엽이 기획한 것으로,
7/10 회의에서 프로토타입(PR #8) 시연과 함께 공유해 다음 사항을 합의로 확인했다:

| 합의 사항 | 회의 발언 요지 |
|---|---|
| 3단계 복사-붙여넣기 플로우가 평가자에게 난해 | "저도 딱 보자마자는 여러 개의 프롬프트 창이구나 이렇게 생각을 했는데" |
| Claude Code 스킬 형태 제출은 더 난해 | "스킬로 하면 평가하시는 분들이 난해하게 생각할 수 있을 것 같다" |
| 평소 AI와 대화하듯 쓰는 단일 플로우가 직관적 | "1번 부분만, 평소 AI랑 작업을 진행하는 것처럼 같은 UI를 만들면 훨씬 직관적이겠네요" |
| 나침반이 실시간으로 나타나야 프로덕트답다 | "언노운 나침반 같은 게 프로덕트처럼 진짜 나타나는… 실시간으로 보여주면 보는 분들도 이해가 확실히 될 것" |
| 실제 모델을 붙여 배포하면 흥미롭다 | "배포를 해서 실제 어떻게 작동하는지까지 보고 싶다면 그것도 흥미로울 것" / OpenRouter 키 공유 합의 |

**개편 방향: 채팅형 단일 플로우 + 실시간 나침반 + OpenRouter 모델 연동.**
단, 기존 수동 플로우는 제거하지 않고 폴백으로 접어둔다 — README의 설계 원칙
("프롬프트 자체가 도구, API 키 불필요")을 버리는 게 아니라 그 위에 편의 계층을 얹는 것.

## 2. 수정 사항 목록

### P0 — 필수

| # | 항목 | 내용 | 회의 발언 요지 |
|---|---|---|---|
| 1 | **채팅형 단일 입력 UI** | Step 2(프롬프트 복사)·Step 3(JSON 붙여넣기)를 기본 화면에서 숨긴다. 사용자는 평소처럼 작업 설명 하나만 입력 → 진단 프롬프트 생성 → 모델 호출 → JSON 파싱 → 시각화가 내부에서 자동 진행 | "평소 AI랑 작업을 진행하는 것처럼 같은 UI" |

| 3 | **실시간 나침반 2-패널 레이아웃** | 좌: 채팅/입력 패널, 우: 4분면 매트릭스. 응답 도착 시 매트릭스 즉시 갱신 | "프로덕트처럼 진짜 나타나는" |

## 3. 목표 UX 플로우

### Before (현행, 3-step 복붙)

```
[1. 작업 설명 입력] → [2. 진단 프롬프트 생성·복사] → (외부 AI에 붙여넣기)
                    → [3. JSON 응답 붙여넣기] → [시각화]
```
사용자가 도구와 외부 AI 사이를 두 번 왕복. 평가자 첫인상: "여러 개의 프롬프트 창".

### After (개편, 채팅 1회 입력 → 실시간 나침반)

```
[작업 설명 입력 + 전송] ──(내부: 프롬프트 생성 → OpenRouter 호출 → JSON 파싱)──▶ [나침반 갱신]
```

## 4. 기술 설계

- **단일 정적 HTML 유지** (`tool/index.html` 하나, 의존성 0). 빌드 도구 도입 없음.
- **OpenRouter 호출**: `fetch POST https://openrouter.ai/api/v1/chat/completions`
  - 헤더: `Authorization: Bearer <키>`, 바디: `{ model, messages: [{role:"user", content: buildPrompt(task)}] }`
  - 브라우저에서 직접 호출 가능(CORS 허용) → 서버리스 배포 성립
- **BYOK 키 관리**: 설정(⚙)에서 입력 → `localStorage` 저장. 소스·저장소에 키를 절대 넣지 않는다 (푸시 시 시크릿 스캐닝/제공자 차단도 회피)
- **기존 로직 재사용**: `buildPrompt()`(진단 프롬프트), `extractJSON()`(응답에서 JSON 추출), `render()`(4분면 렌더링)는 그대로 두고 **호출부만 추가** — 수동 모드 폴백이 공짜로 유지된다
- **실패 처리**: JSON 파싱 실패 시 "JSON만 다시 출력해줘" 재시도 1회 → 그래도 실패하면 원문 응답 표시 + 수동 모드 안내. 키 없음/401 → 수동 모드 안내
- **모델 기본값**: GLM 계열 저비용 모델(설정에서 변경 가능한 문자열 하나로 관리)

## 5. 접근성·디자인 원칙 승계 (변경 없음)

- 4분면 팔레트: CVD(색각 이상) 분리도·명도 대비 검증 통과한 기존 4색 유지
- 색상만으로 의미 전달 금지 — 전 사분면 텍스트 라벨 병행
- 라이트/다크 모드 (`prefers-color-scheme`) 유지
- 시스템 sans 서체, 텍스트는 텍스트 토큰 색상만 사용
